Gestionar proyectos de desarrollo con metodología ágil, QA y versiones.

- Historia de Usuario
- Tarea Técnica
- Bug
- Sub-tarea

- Story Points
- Componentes
- Severidad (bugs)
- Fix Version
- Criterios de Aceptación

3. Code Review
4. QA
5. Done

- Smart commits (Git)

- Notificación por bugs críticos

- Confluence: requisitos y diseño
